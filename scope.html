Scope / Specifications

Denne siden er laget for utviklere og partnere: hva som må bygges for å automatisere domene → repo → deploy → SSL → oppdateringer via kundens GPT.

1) Onboarding

Domene-input → DNS-sjekk (A/AAAA/CNAME) → firmainfo (brief) → opprett repo fra template i vår GitHub-org → initial commit.

2) Deploy + SSL

GitHub Actions (eller runner) → atomic deploy til VPS → Nginx vhost → Let’s Encrypt (ACME) → healthcheck → rollback.

3) Modul-system

Tilvalg (Vipps, analytics, form, chat). Plattformen spør etter nøkler → validerer → lagrer som secrets → patch config → redeploy.

4) BYO-AI (kundens GPT)

Kunden bruker egen AI-konto til å generere endringer. Best praksis: endringer skapes som PR → preview → merge → deploy.

Acceptance criteria (MVP)
1) Kunde kan oppgi domene og få presise DNS-instrukser
2) Repo opprettes automatisk (template) og første deploy skjer uten manuell jobb
3) SSL er automatisk provisionert
4) Deploy skjer konsistent (atomic) + rollback fungerer
5) Dashboard viser status + siste deploy + feil
